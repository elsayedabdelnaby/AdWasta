# Research Camoufox Firefox anti-detect browser

## Camoufox: Anti-Detect Firefox Browser — Research Report (July 2026)

**1. What it is / maintainer.** Camoufox is an open-source anti-detect browser built by compiling Firefox from source with C++-level patches, targeted at web scraping and AI-agent automation. Repo: https://github.com/daijro/camoufox (created 2024-07-26, per `gh api repos/daijro/camoufox`). Originally solo-maintained by daijro; on **2026-01-10** daijro announced stepping down as primary maintainer after a personal/medical hiatus, handing active development to Clover Labs while retaining repo ownership ("The Camoufox core repo remains owned by Daijro. We did not buy it, we are simply maintaining it.") — https://github.com/daijro/camoufox/discussions/452. Docs site: https://camoufox.com/.

**2. Architecture vs. Chromium approaches.** Camoufox does not use CDP. It patches Firefox's own automation protocol, **Juggler**, giving Playwright "its own isolated 'copy' of the page to work with... the real page is completely unaffected" — https://camoufox.com/stealth/. Spoofing happens "at the C++ implementation level," so hijacked objects "appear native" with "no JavaScript hijacking to be detected" (same source). This is a compile-time/native-code approach, structurally different from Patchright/rebrowser/stealth-plugins, which patch at the CDP or JS-injection layer post-launch and remain detectable via property-descriptor/`toString` inspection or CDP leak signatures (per the same page, contrasting JS-based solutions).

**3. Fingerprinting protections.** Per https://camoufox.com/fingerprint/, the config surface covers: Navigator properties, Cursor Movement, Fonts, Screen, Window, Document, HTTP Headers, Geolocation & Intl, WebRTC IP, WebGL, Media & Audio, Voices, Addons, and Misc. Cursor movement is a human-like trajectory algorithm "originally from riflosnake's HumanCursor... rewritten in C++ and modified for more distance-aware trajectories" (https://camoufox.com/stealth/). Device data is auto-populated via BrowserForge to match real-world traffic distributions (https://camoufox.com/python/browserforge/). Explicit limitation stated in the docs: Camoufox does **not** support injecting Chromium-style fingerprints, since V8-specific JS engine behavior can't be convincingly spoofed against competent WAFs (https://camoufox.com/fingerprint/).

**5. License.** MPL-2.0, confirmed via GitHub API (`license.spdx_id: "MPL-2.0"`) and a LICENSE file present in the repo. Notably, the separate Python wrapper package on PyPI is licensed **MIT**, not MPL-2.0 (https://pypi.org/project/camoufox/) — a distinction worth flagging for compliance review since the compiled browser and the launcher library carry different obligations.

**6. Playwright integration.** PyPI package `camoufox` (latest 0.5.4, released 2026-07-16, author daijro.dev@gmail.com) is described as a "Lightweight wrapper around the Playwright API to help launch Camoufox" — https://pypi.org/project/camoufox/, https://camoufox.com/python/. Install via `pip install -U camoufox`, then fetch the binary with `camoufox fetch` / `python3 -m camoufox fetch`. It supports both sync and async Playwright APIs and auto-injects fingerprints, geolocation/timezone/locale (proxy-consistent). It is not simply Playwright's stock Firefox channel — it launches the custom Camoufox binary through the patched Juggler protocol. A separate `camoufox-connector` package adds a WebSocket bridge for non-Python (e.g., Node/Playwright) access (https://libraries.io/pypi/camoufox-connector).

**7. Limitations / detection reports.** No independent academic detection audit was found. Third-party blog claims (unverified, no dates given) assert passing CreepJS, BrowserScan, and Fingerprint.com checks and bypassing Cloudflare Turnstile/Interstitial, DataDome, and Imperva — https://roundproxies.com/blog/camoufox/. The same source acknowledges sites can still block Camoufox, recommending "virtual headless mode," human mouse movement, and residential proxies as mitigations. Camoufox's own docs concede unresolved fingerprint-consistency risk across GPU/OS/hardware combos and the built-in inability to fake Chromium/V8-specific engine fingerprints.

**Verdict.** Camoufox is architecturally more robust than JS/CDP-patch approaches (Patchright, rebrowser, stealth plugins) because its spoofing lives in compiled C++ Firefox internals rather than runtime-patched JS/CDP surfaces that leave forensic traces — a real trust advantage for a SaaS doing browser-based social posting. But it is a single-maintainer-turned-small-org project that went dark for ~10 months in 2025 and only stabilized under Clover Labs from January 2026; it self-labels as experimental even in its latest (July 2026) releases. For production use, treat it as promising but not yet enterprise-hardened: pin exact versions, monitor the CloverLabsAI/daijro repo split, and keep a Chromium-based fallback (Patchright/rebrowser) path rather than single-sourcing anti-detect infrastructure on it.
