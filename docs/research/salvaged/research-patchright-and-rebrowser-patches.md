# Browser Automation Anti-Detection: Patchright & rebrowser-patches (research as of 2026-07-17)

## Patchright

**Identity & packages.** The project is Kaliiiiiiiiii-Vinyzu/patchright on GitHub, split across three repos: the main driver (github.com/Kaliiiiiiiiii-Vinyzu/patchright), patchright-python, and patchright-nodejs. Maintainers are listed as Vinyzu (active) and Kaliiiiiiiiii (co-maintainer) (https://pypi.org/project/patchright/). It ships on both registries: PyPI `patchright` v1.61.2, released July 5, 2026 (https://pypi.org/project/patchright/), and npm `patchright` v1.61.1, published June 23, 2026 per the npm registry API (https://registry.npmjs.org/patchright, https://www.npmjs.com/package/patchright). License is Apache-2.0, confirmed both in the PyPI/npm metadata and the repo's LICENSE file (https://github.com/Kaliiiiiiiiii-Vinyzu/patchright/blob/main/LICENSE).

**Activity.** Via GitHub API: main repo 3,802 stars, last push 2026-07-15, 4 open issues; patchright-python 1,429 stars, last push 2026-07-16, 1 open issue; patchright-nodejs 742 stars, last push 2026-07-16, 0 open issues ŌĆö i.e., commits within the last two days of this report, tracking Playwright's own release cadence closely.

**Detection vectors patched** (per README, https://github.com/Kaliiiiiiiiii-Vinyzu/patchright/blob/main/README.md): (1) **Runtime.enable leak** ŌĆö called "the biggest patch," avoided by executing JS in isolated ExecutionContexts instead of calling Runtime.enable; (2) **navigator.webdriver** ŌĆö via `--disable-blink-features=AutomationControlled` and removing `--enable-automation`; (3) **Console.enable leak** ŌĆö fixed by disabling the Console API entirely; (4) **command-line flag leaks** ŌĆö removes `--disable-component-update`, `--disable-default-apps`, `--disable-extensions`, `--disable-popup-blocking`, which otherwise reveal a stealth driver; (5) **closed shadow-root interaction** ŌĆö normal locators/XPath work inside closed shadow DOM. Chromium-only (no Firefox/WebKit). The README also flags a theoretical initScript timing-attack vector it doesn't defend against, noting "no antibot currently checks for this kind" of attack.

**Independent testing (2025-2026).** Multiple write-ups (ZenRows https://www.zenrows.com/blog/patchright; RoundProxies https://roundproxies.com/blog/patchright/) claim passes against Cloudflare, Kasada, Akamai, DataDome, Fingerprint.com, CreepJS, Sannysoft, Incolumitas, IPHey, Browserscan, Pixelscan ŌĆö but one source notes CreepJS headless-detection score drops from 100% to only ~67%, i.e., improved, not eliminated. A dated, methodologically explicit benchmark (ianlpaterson.com, published May 13 2026, updated July 12 2026, https://ianlpaterson.com/blog/anti-detect-browser-benchmark-patchright-nodriver-curl-cffi/) ran 31 real-world targets three times each: Patchright scored 25 OK / 3 gated / 3 blocked, versus 24 OK for vanilla Playwright ŌĆö only a one-site net improvement ŌĆö and the author concludes that using Chrome's real `channel=chrome` binary mattered more than the patches themselves. A 2026 comparison (DataResearchTools, https://dataresearchtools.com/patchright-vs-rebrowser-patches-stealth-playwright-patches-compared-2026/) recommends Patchright for most teams for its lower setup cost and Python support, though it also claims rebrowser-patches has a marginal edge specifically against Cloudflare Bot Management. Open issue #211 ("Patchright on arm64 arch Gets Caught," opened 2026-05-31) shows unresolved architecture-specific detection gaps (https://github.com/Kaliiiiiiiiii-Vinyzu/patchright/issues).

**Verdict:** Actively maintained, permissively licensed, healthy community, fast upstream tracking. Production-usable as a baseline stealth layer for Chromium, but independent 2026 benchmarks show it is an incremental improvement, not a silver bullet ŌĆö pair with real Chrome channel binaries, residential proxies, and fingerprint rotation.

## rebrowser-patches

**Identity & mechanism.** Repo rebrowser/rebrowser-patches, packages `rebrowser-playwright`/`rebrowser-playwright-core` and `rebrowser-puppeteer`/`rebrowser-puppeteer-core`. The core claim, per the rebrowser blog (https://rebrowser.net/blog/how-to-fix-runtime-enable-cdp-detection-of-puppeteer-playwright-and-other-automation-libraries): automation libraries call CDP `Runtime.enable` to receive execution-context events; this triggers `Runtime.consoleAPICalled` events and exposes `ExecutionContextId`s for every frame/worker, letting a few lines of page JS detect CDP control ŌĆö a technique "used by all major anti-bot software such as Cloudflare, DataDome." The patch disables automatic Runtime.enable and instead uses isolated worlds / addBinding or quick enable-disable sequences to obtain contexts without the persistent signal.

**Activity ŌĆö stale.** GitHub API confirms: 1,398 stars, last push 2025-05-09T05:24:53Z, 33 open issues, **license: none detected** by GitHub (https://github.com/rebrowser/rebrowser-patches) ŌĆö despite the npm packages' package.json declaring Apache-2.0 (confirmed via https://registry.npmjs.org/rebrowser-playwright-core and https://registry.npmjs.org/rebrowser-puppeteer-core), a discrepancy worth flagging before adoption. Latest releases: `rebrowser-playwright` 1.52.0 (May 9, 2025) and `rebrowser-puppeteer` 24.8.1 (May 8, 2025) ŌĆö no releases or commits since, per https://github.com/rebrowser/rebrowser-patches/releases and /commits/main.

**Effectiveness concerns.** Open issue #114 ("Latest patch release 1.0.19 doesn't pass all the tests from rebrowser-bot-detector with puppeteer," opened 2025-07-08, still open, no maintainer reply as of this research) ŌĆö the maintainers' own detection test suite reportedly fails against their newest patch, unaddressed for over a year (https://github.com/rebrowser/rebrowser-patches/issues/114).

**Verdict:** Effectively unmaintained (14+ months with zero commits as of 2026-07-17), pinned to mid-2025 Playwright/Puppeteer versions, growing issue backlog, and unresolved self-reported test failures. License status is ambiguous/conflicting between GitHub and npm metadata. The Runtime.enable-avoidance concept is sound (Patchright independently implements the same idea), but this specific project is not recommendable for new 2025-2026 production work without accepting stale upstream pins and manual patch maintenance.
